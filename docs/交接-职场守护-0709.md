## 本次做完（Karen 三个实测反馈）

### 1. AI 教练中英混杂 → 已修（`api/coach.js`）
- **现象**：Karen UI 设中文、用中文问"这算霸凌吗"，AI 却用英文回复。
- **根因**：`SYS_ZH` / `SYS_EN` 系统提示**没强制输出语言**，Qwen(qwen-plus) 会漂移到英文。前端 `lang:LANG` 传参是对的（`app.html:865`）。
- **修复**：
  1. 两个系统提示**顶部**加强制语言锁（"无论用户用什么语言、无论上文什么语言，每句都必须用简体中文/English only，绝不混杂"）。
  2. 末尾再把语言提醒**并进最后一条 user 消息**（模型对最近指令权重更高，防历史语言污染；并进 user 而非新加 system，避免个别兼容接口对"非首位 system"挑剔）。
- **线上验证**：`curl .../api/coach {lang:zh, 中文问}` → **纯中文回复**，调性达标（先接情绪"你愿意说出来已经很勇敢了"、不急着下定义、正向肯定）。

### 2. 时间线 KPI 灰又小看不清 → 已修（`app.html` 内联 CSS）
- `.kpi-n`（数字）：`24px → 30px`，加 `color:#fff` 纯白。
- `.kpi-l`（标签 条数/有证据/完整度）：`10px 灰 #8a8070 → 12px 白 rgba(255,255,255,.82) + 600`。
- 元凶：深黑底(`--ink #0E0C0A`)上用灰字对比度太低。

### 3. 支付二维码去外部依赖 → 已改（`app.html`）
- **原来**：用 `api.qrserver.com`（德国图床）把 `code_url` 渲染成二维码 `<img>`。**风险**：目标用户在国内，该外部服务大概率慢/被墙 → 二维码永远转圈、**用户付不了款却无任何提示**。
- **现在**：内联 `qrcodejs`(davidshimjs·~20KB) 本地 `canvas` 渲染（同时生成 `<img>` 长按可识别）；库异常才回退外部图床。
- **验证**（Playwright 本地）：真实调 Lumee 后端拿到真 `weixin://...` 码 → 本地渲染成 canvas(210×210·21807 暗像素) → 网络面板**零 `api.qrserver.com` 请求**，只有 `mylumee.cn` 建单+轮询。
